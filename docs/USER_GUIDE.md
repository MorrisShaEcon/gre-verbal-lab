# GRE Verbal Lab v2.2.0 使用指南

## 1. 第一次打开

不需要注册，也不需要先导入词库。私人版本会直接加载 2,535 个核心词，并为当天生成
学习计划。词汇目录和个人学习记录分开保存；以后更新数据库不会清空已经形成的复习历史。

词库总量不等于当天可学数量。只有完成义项去重、词性核对、可信 IPA、同反义字段查证、
精确或经编辑审核批准的义项对齐，以及合格来源例句的词义才会进入正式学习队列。词义库只展示
这一正式层；其余条目保留在后台素材库，等待后续校对。

## 2. 先设定今天想学多少新词

首页的“今天想新学多少个词？”可以设为 1–200。这个数字表示今天的新词总目标，不是每次
重新生成一整批。系统会扣除今天已经完成的新词，显示今天还剩多少、预计需要多少学习日，
以及按当前速度完成正式词库的日期。中途修改目标不会重复计算已经完成的部分。

完成当前批次后，可以直接点击“继续学新词”并指定追加数量。追加批次会立即保存到今天的
计划，因此刷新或关闭页面后，未完成的词和顺序仍会保留；想多学时不受最初目标限制。

## 3. 今日 70/30 队列

每日新词默认由两层组成：

- 70% 重点层：综合核心/补充词表交集、本地材料出现次数和已有遗忘证据；
- 30% 长尾层：低覆盖、生僻和长期未见词，防止只会常见词。

两层内部使用日期和数据库版本生成随机种子。当天刷新后顺序保持一致，第二天重新抽样，
不会沿用 Excel 的 A-Z 行号。到期复习独立加入，不受 70/30 比例限制。未通过内容门槛的
词不会为了凑数进入队列。

## 4. 如何完成一张卡

1. 先看单词、词性和音标，也可以播放发音。
2. 在四个中文释义中选择最准确的一项；键盘可直接按 1–4。
3. 系统立即判断正误，不需要你再选择“认识、模糊或忘记”。
4. 作答后再查看正确释义、英文解释、同义词、反义词、例句、来源和安排原因。
5. 点击“下一个词”继续。

三个错误选项本身也必须通过正式内容门槛。292 个可出题义项各自绑定了三个经过
编辑审定的、相同词性的具体易混义项，候选来自 296 个正式义项构成的选项池；其中 4 个
副词义项只会充当干扰项，不会成为题干。系统不会把纯粹拼写相近冒充为语义混淆，也会排除
同一 OEWN/CILI 概念、双向同义词和可能同时正确的近重复释义。
作答后会显示三个易混词各自的中文义和逐义项编辑辨析理由。

## 5. 系统如何判断是否掌握

- **答错：** 记录为未掌握，缩短复习间隔，并在本轮稍后再出现一次。
- **第一次答对：** 只记为正在学习。四选一存在 25% 猜中概率，一次正确不足以证明掌握。
- **多次答对：** 结合历史遗忘和间隔，逐步判断为记住或熟练。
- **答得慢：** 如果设置中允许，反应时间会作为辅助证据；它不会单独决定结果。
- **无计时模式：** 可关闭反应时间判断，系统只根据正确性和历史记录安排复习。

这些内部判断会转换成复习间隔，但核心流程不会要求你手动给自己打分。

## 6. 发音

播放按钮优先使用 Lingua Libre 或 Wikimedia Commons 的开放许可真人录音。卡片会显示
口音、来源、录音者和许可证，并可选择 0.8× 慢速或 1× 原速。

音标只展示开放词典原生 IPA 或带固定修订页的人工核验 IPA。CMUdict 的 ARPAbet 转写仍可
作为后台发音证据，但没有可靠音节边界，不能单独让一个词进入正式学习。

如果词库没有内置录音，第一次点击会尝试从 Commons 查找精确匹配并保存来源信息；找到后
再次点击即可播放。首次查询需要联网，播放过的媒体可由浏览器按需缓存。

没有合格真人录音时，应用才会提供“系统合成备用音”，并明确标注为合成语音。它不是词典
真人录音，也不会被包装成真人发音。

## 7. 例句与本地机经原文

正式学习队列中的每个义项必须至少有一个合格例句。系统同时检查：

1. 能否追溯到具体来源；
2. 当前私人或公开版本是否有权使用；
3. 例句是否真正表达这张卡正在考的中文义项；
4. 目标词或允许的词形变化是否实际出现在句中。

“来源可信”和“义项匹配”不是一回事。例如一个词典例句可能真实存在，却对应同一单词的
另一个意思；这种例句不会进入正式队列。

例句标签包括：

- `开放词典例句`：保留来源与许可证；
- `GRE 官方`：只有取得当前使用方式所需授权时才能保存原文；
- `影视语境`：只有取得台词或字幕的使用授权时才能保存原文；
- `原创 GRE 风格`：项目自行撰写的补充练习，不是真题，也不冒充真题；
- `私人引用`：只在你的设备保存来源、题号、页码、集数或时间点等信息。

私人版还可读取本地生成的 GRE 机经/练习题索引。当逐题语义审核确认题目表达了当前卡片
的准确义项，作答后会出现“本地 GRE 机经题库原文”，可展开查看文章、题干、选项、答案、
PDF 页码和命中位置。题库数量、内部题号和源文件信息只保存在本地。

两种状态必须区分：

- `已确认当前义项`：逐题复核确认原文中的词就是卡片正在学习的意思；
- `仅词形命中`：题目里出现了同一个拼写，但尚不能证明是当前义项，只作为检索线索。

没有匹配时，应用会明确说明当前义项没有通过审核的本地题目，不会拿开放词典例句冒充
机经，也不会把机经标成 ETS 官方真题。

## 8. 版权与私人数据边界

公开 GitHub 版本不会捆绑 ETS 真题原文、答案、解析，也不会复制没有再发布许可的电影或
美剧台词。购买书籍、订阅流媒体或能在词典 App 里查看内容，并不自动等于可以公开转载。

你可以在本地记录自己合法持有材料的题号、页码、集数、时间点和个人笔记；如使用本地原文，
它只保存在私人数据库中，不会进入公开词库或 GitHub Release。私人导入功能是本地学习工具，
不是版权授权工具。

## 9. 词义网络与词汇错因

“正式词义库”按义项展示关系、来源、对齐状态和用户笔记。手动修改保存在个人 overlay，
数据库升级不会覆盖，也不会自动发布。

“词汇错因”记录题目引用、错因和个人提醒。定义题的选项、所选答案、正确答案、反应区间和
干扰义项会写入事件日志。之后再次练习同一目标义项时，系统会优先放回你曾经选错的那个
具体义项；正误、遗忘次数和反应时间同时用于调整目标词的复习间隔。

“本地材料遇见次数”只是私人材料中的派生统计，不是 ETS 官方词频。

## 10. GRE 模拟题错题模块

“错题回炉 → GRE 模拟题错题”是为今后的截图工作流预留并已可用的本地模块。可以：

1. 添加截图，并填写题型、来源、套题/题号、文章、题干、选项、你的答案和正确答案；
2. 选择错因（词汇、逻辑、定位、时间、粗心等），记录陷阱与改进动作；
3. 关联具体词义，使相关词进入强化队列；
4. 到期后先重新作答，系统判分后才显示答案和解析，再根据难度安排下一次复习；
5. 导出不含截图的整理 JSON，或用完整备份保留包括截图在内的私人资料。

截图只保存在当前浏览器，不会自动上传。目前应用不调用外部 OCR：以后你把 GRE 错题截图
发给 Codex，我会识别并整理成这个模块可导入的结构；你也可以在页面里手动补充或校正。

## 11. 设置、备份和高级导入

“设置与备份”可以修改考试日期、每日新词数、到期复习上限、真人录音速度，以及是否使用
反应时间辅助判断。完整 JSON 备份包含个人学习状态，应像个人文件一样妥善保管。

XLSX/CSV 导入位于“高级数据管理”，只用于迁移额外个人词表或本地材料，不是首次使用的
必经步骤。导入内容默认只保存在当前浏览器。

“清空学习记录”保留默认数据库，只删除当前浏览器中的掌握度、错题和历史。
